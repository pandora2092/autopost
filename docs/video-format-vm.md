# Видео не проигрывается в VM, кнопка «Поделиться» неактивна

Если в VM (Android-x86) **видео не проигрывается** (в галерее появляется «Can't play this video.» или чёрный экран, фото тоже не открываются), ниже — пошаговая настройка VM/папки и перекодирование.

---

## Пошаговая настройка VM и папки (кодеки, графика)

Делать по порядку. После каждого шага проверять в VM: открыть галерею → воспроизвести тестовое видео или фото.

### Шаг 1. Видео-драйвер VM (KVM/libvirt)

От того, какой виртуальный видеоадаптер видит Android, зависит работа декодера и отрисовки.

1. Останови VM (если запущена).
2. Посмотри текущую конфигурацию:
   ```bash
   virsh dumpxml ИМЯ_VM | grep -A5 "<device.*video\|<model type"
   ```
   Будет что-то вроде `<model type='qxl'/>` или `<model type='virtio'/>` или `vga`.
3. Отредактировать конфиг VM:
   ```bash
   virsh edit ИМЯ_VM
   ```
   Найди блок `<devices>` и внутри — секцию `<video>` и `<graphics>`. В `<video>` должен быть дочерний элемент `<model type='...'/>`.
4. Попробуй по очереди (сохрани, запусти VM, проверь воспроизведение в галерее):
   - **VirtIO GPU:** `<model type='virtio'/>` (часто лучше для Android в KVM).
   - Если уже virtio и не помогает — **QXL:** `<model type='qxl'/>` (иногда стабильнее на старых образах).
   - Либо **VGA:** `<model type='vga'/>` / `<model type='cirrus'/>` в зависимости от того, что разрешает образ.
5. Перезапусти VM после смены и снова проверь галерею.

Если редактируешь **шаблон** (template), после смены видео все новые клоны будут с новой настройкой. Уже созданные VM правь через `virsh edit ИМЯ_КЛОНА`.

### Шаг 2. ОЗУ и CPU

В том же `virsh edit ИМЯ_VM` проверь:
- **Память:** не меньше 2048 МБ, лучше 3072–4096.
- **CPU:** минимум 2 ядра.

Сохрани, перезапусти VM.

### Шаг 3. Папка для медиа (Download vs DCIM)

По умолчанию бэкенд кладёт видео в `/sdcard/Download`. На части прошивок галерея/Instagram лучше подхватывают файлы из `/sdcard/DCIM`.

**Вариант A — только для теста вручную:**
```bash
adb -s IP:5555 push тест.mp4 /sdcard/DCIM/
adb -s IP:5555 shell chmod 644 /sdcard/DCIM/тест.mp4
adb -s IP:5555 shell am broadcast -a android.intent.action.MEDIA_SCANNER_SCAN_FILE -d "file:///sdcard/DCIM/тест.mp4"
```
Открой галерею в VM и проверь, воспроизводится ли файл. Если в DCIM всё ок, а в Download — нет, переходи к варианту B.

**Вариант B — чтобы бэкенд всегда использовал DCIM:**  
Перед запуском бэкенда задай переменную и перезапусти сервис:
```bash
export REMOTE_MEDIA_DIR=/sdcard/DCIM
# затем запуск backend-nest (npm run start и т.п.)
```
Тогда при публикации поста файл будет пушиться в `/sdcard/DCIM`, с теми же chmod и медиа-сканером.

### Шаг 4. Перекодирование видео в H.264 baseline

Если после шагов 1–3 видео в галерее по-прежнему не воспроизводится или фото чёрные — пробуй максимально совместимый формат (см. раздел «Формат и кодек» ниже). Если после смены видео-драйвера и папки заработало — перекодирование можно использовать по желанию для уменьшения размера или совместимости.

---

## Что сделать (формат, права, прошивка)

### 1. Формат и кодек

На Android-x86 (и многих эмуляторах) надёжно работает только **H.264 (AVC)** в контейнере **MP4**.

- **Используйте:** MP4 с видеокодеком H.264, звук AAC.
- **Избегайте:** HEVC (H.265), VP9, AV1, нестандартные разрешения/профили.

Перекодирование через ffmpeg (на хосте):

```bash
ffmpeg -i исходное.mp4 -c:v libx264 -profile:v high -level 4.0 -c:a aac -movflags +faststart output.mp4
```

Для рилсов обычно подходит разрешение до 1080p и соотношение сторон 9:16.

### 2. Медиа-сканер после загрузки файла

После `adb push` файл может быть в `/sdcard/Download`, но система ещё не добавила его в медиабазу — из-за этого галерея и Instagram иногда «не видят» файл или считают его нечитаемым. В коде публикации уже вызывается медиа-сканер после push; при ручной загрузке можно запустить его вручную:

```bash
adb -s IP:5555 shell am broadcast -a android.intent.action.MEDIA_SCANNER_SCAN_FILE -d "file:///sdcard/Download/имя_файла.mp4"
```

Подставьте свой ADB-адрес VM и имя файла. Через несколько секунд файл должен появиться в галерее и стать доступным для выбора в Instagram.

### 3. Права доступа на файл

После `adb push` файл может создаваться с правами, при которых приложения (Галерея, Instagram) не могут его прочитать. На устройстве выполните:

```bash
adb -s IP:5555 shell chmod 644 /sdcard/Download/имя_файла.mp4
```

В коде публикации после push уже вызывается `chmod 644`; при ручной загрузке сделайте это вручную.

### 4. «Can't play this video» в галерее Android

Если при открытии видео в **галерее VM** появляется сообщение **«Can't play this video.»** — системный декодер в этой прошивке не справляется с файлом (профиль/уровень H.264, разрешение или контейнер). Права и медиа-сканер тут не помогут.

**Что сделать:**

1. **Максимально совместимое перекодирование** — H.264 **baseline**, низкое разрешение, простой контейнер:
   ```bash
   ffmpeg -i исходное.mp4 -c:v libx264 -profile:v baseline -level 3.0 -vf "scale=640:480:force_original_aspect_ratio=decrease" -b:v 800k -maxrate 800k -bufsize 400k -c:a aac -b:a 128k -movflags +faststart -pix_fmt yuv420p output.mp4
   ```
   Если такая команда даёт ошибку по фильтру scale — используй упрощённый вариант (фиксированная высота):
   ```bash
   ffmpeg -i исходное.mp4 -c:v libx264 -profile:v baseline -level 3.0 -vf "scale=-2:480" -b:v 800k -c:a aac -b:a 128k -movflags +faststart -pix_fmt yuv420p output.mp4
   ```
   Для рилса 9:16:
   ```bash
   ffmpeg -i исходное.mp4 -c:v libx264 -profile:v baseline -level 3.0 -vf "scale=-2:854" -b:v 800k -c:a aac -b:a 128k -movflags +faststart -pix_fmt yuv420p output.mp4
   ```

2. **Тестовый ролик** — проверить, воспроизводится ли на VM **вообще** какое-то видео. На хосте (должен быть установлен ffmpeg):
   ```bash
   cd /home/ann/workspace/new/scripts
   ./make-test-video.sh
   ```
   Получится файл `test-video-vm.mp4`. Залейте его на устройство, откройте в галерее. Если и он не проигрывается — в образе Android-x86/Bliss, скорее всего, нет рабочего программного декодера или нужен другой образ с поддержкой медиа.

3. **Смена прошивки** — часть образов Android-x86 поставляется без полного набора кодеков. Рекомендуемые сборки с учётом Instagram и кодеков: [android-vm-builds.md](android-vm-builds.md) (Bliss OS, AndEX и др.).

### 5. Проверка кодека на хосте

Убедиться, что файл именно H.264:

```bash
ffprobe -v error -select_streams v:0 -show_entries stream=codec_name,profile -of default=noprint_wrappers=1 ваш_файл.mp4
```

Ожидаемый вывод: `codec_name=h264`, профиль желательно `baseline` или `main` для максимальной совместимости.

---

**Итог:** при H.264 проверьте права (chmod 644), при необходимости перекодируйте в baseline и/или положите файл в DCIM и заново запустите медиа-сканер.
