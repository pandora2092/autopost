# Первый запуск: создание первой VM по шагам

Выполняйте шаги по порядку на хосте с KVM, где уже есть шаблон **android-template**.

---

## Шаг 0. Подготовка (один раз)

Убедитесь, что скрипты исполняемые и шаблон есть в libvirt:

```bash
cd /home/ann/workspace/new
chmod +x scripts/*.sh
virsh list --all
```

В списке должен быть домен **android-template** (состояние не важно). Если имя другое — задайте его при запуске бэкенда: `export VM_TEMPLATE_DOMAIN=ваше-имя`.

---

## Шаг 1. Запуск бэкенда и фронтенда

**Терминал 1 — бэкенд (NestJS):**

```bash
cd /home/ann/workspace/new/backend-nest
export VM_TEMPLATE_DOMAIN=android-template
npm run start:dev
```

Дождитесь строк: `Scheduler started`, `Server running at http://localhost:3000`.

**Терминал 2 — фронтенд (React):**

```bash
cd /home/ann/workspace/new/frontend
npm run dev
```

Откройте в браузере: **http://localhost:5173**

---

## Шаг 2. Прокси (опционально)

Если для первой VM прокси не нужен — переходите к шагу 3.

- В панели: **Прокси** → «Добавить прокси».
- Укажите хост, порт, при необходимости логин/пароль (тип SOCKS5 подставляется сам).
- Сохраните.

---

## Шаг 3. Создание VM

- В панели: **Виртуальные машины** → «Создать VM».
- **Имя VM:** латиница без пробелов, например `instagram-vm-01`.
- **Прокси:** выберите из списка или оставьте «— Прокси (опц.) —».
- Нажмите **Создать**.

Бэкенд вызовет скрипт клонирования; через несколько секунд в списке появится новая VM со статусом **stopped** и своим MAC.

---

## Шаг 4. Запуск VM

- В карточке созданной VM нажмите **Старт**.
- Подождите 1–2 минуты, пока Android загрузится.
- Статус сменится на **running**.

---

## Шаг 5. Узнать IP (и автоматически выставить Android ID)

- В карточке VM нажмите **«Узнать IP»**.
- Бэкенд запросит IP у libvirt, сохранит ADB-адрес как `IP:5555` и **автоматически установит уникальный Android ID**.
- Если IP получен — появится сообщение «IP и Android ID установлены».
- Если IP не получен — подождите 1–2 минуты после запуска VM и нажмите **«Узнать IP»** снова. При необходимости можно ввести адрес вручную: **«Ввод ADB»** → `IP:5555` → **Сохранить ADB**.

---

## Шаг 6. Дальше (профиль и Instagram)

- **Профили** → «Добавить профиль» → выберите эту VM, при желании укажите логин Instagram.
- Установите приложение Instagram на VM (вручную в шаблоне или через `adb -s IP:5555 install ...apk`).
- В профиле нажмите **«Открыть экран»** — откроется страница с командой scrcpy; выполните её на ПК и войдите в Instagram вручную на экране VM.

После этого VM готова к использованию в планировании публикаций.
